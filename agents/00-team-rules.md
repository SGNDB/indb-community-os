# INDB Community OS — Team Rules

You are working on INDB Community OS.

Project vision:
A digital community platform for Nouadhibou focused on posts, memories, ideas, profiles, contribution, community pride, and civic participation.

Core rules:

- Do not break Arabic/French/English translations.
- Arabic must remain RTL.
- French and English must remain LTR.
- Do not expose secret keys.
- Never put Supabase service role key in frontend.
- Do not remove existing working features.
- Do not redesign unrelated pages.
- Use Supabase securely.
- Use real data, not mock data.
- Preserve current locale in all routes.
- If touching database, check RLS.
- If touching uploads, check Supabase Storage policies.
- If touching UI, test mobile and desktop.
- Every visible button should work or show a clear coming-soon message.
- Explain what files you changed.
- Explain what needs manual setup in Supabase or Vercel.

Agent coordination:

- Product Lead decides direction.
- UI/UX handles design only.
- Frontend handles user features.
- Supabase Backend handles database and data.
- Media handles images/videos.
- Security/API reviews permissions, API keys, and sensitive flows.
- Admin/Credits handles admin dashboard and contribution score.
- Prompt Engineering handles AI prompts.
- QA tests before deployment.

Do not work outside your role unless clearly requested.

If a task touches another agent’s area, say which agent should review it.
